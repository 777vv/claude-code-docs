## 1.1 一句话定位

**OpenClaw** 是一个**开源、本地运行**的**个人 AI 助手平台**——你在自己已经用的聊天工具（飞书、Telegram、微信群……）里跟它说话，它能**真的去执行任务**：收邮件、发消息、改文件、调 API、跑脚本、自动化浏览器、定时巡检、甚至帮你写代码。

它由 PSPDFKit 创始人 Peter Steinberger 发起，2026 年初开源后 72 小时收获 6 万 GitHub Star，到本文写作时已超过 21 万。社区给它的吉祥物起名"Molty"，是一只太空小龙虾 🦞。

> **关键词**：本地优先（Local-First）· 多渠道（Multi-Channel）· 代理式（Agentic）· 开源（MIT）

## 1.2 它和 ChatGPT / Claude Code / Codex 哪里不一样？

新手最容易混淆这四类工具。一张表分清：

| 维度 | ChatGPT 网页版 | Claude Code | Codex | **OpenClaw** |
|---|---|---|---|---|
| **类别** | 聊天 | AI 编程助手 | AI 编程助手 | **个人 AI 代理平台** |
| **跑在哪** | OpenAI 云服务器 | 你的终端 / IDE | 你的终端 | **你的电脑后台常驻** |
| **用户界面** | OpenAI 网页 | 命令行 / IDE 插件 | 命令行 | **你已用的聊天工具** |
| **主要场景** | 问答、写文案 | 改代码 | 改代码 | **办公自动化 / 信息整理 / 协作 / 也能改代码** |
| **能"自己干活"吗** | ❌ 给你建议 | ✅ 写代码、跑命令 | ✅ 写代码、跑命令 | ✅ **写代码、跑命令、发邮件、抓网页、定时执行……** |
| **数据放哪** | OpenAI 服务器 | 本地+API | 本地+API | **完全本地** |
| **多人共用** | 各用各的账号 | 单机 | 单机 | ✅ **可以做团队机器人** |

简单总结：
- **ChatGPT**：能聊，但不能动手
- **Claude Code / Codex**：能动手，但只能改代码、只能你一个人用
- **OpenClaw**：能动手，能用聊天界面驱动，能定时执行，能多人协作，能管多个 agent 各司其职

::: tip 一个生活化比喻
- **ChatGPT** 像一个**知识渊博的网友**，你问他什么他都答得头头是道，但只能给你建议
- **Claude Code / Codex** 像一个**远程工作的程序员同事**，你给他登录权限，他能直接改代码
- **OpenClaw** 像一个**长期住在你家、有自己钥匙的私人秘书**——能听你随口吩咐、能定时主动办事、能同时帮你和家人/同事处理事
:::

## 1.3 它能做什么（真实场景）

OpenClaw 能力面很广，下面是常见类别。每一类后面会在本站的[实战案例](/openclaw/cases/daily-news)里展开详细教程。

### 信息收集与整理
```
每天早上 8 点，把 V2EX、Hacker News、我关注的 RSS 抓一遍，
摘要后推到飞书机器人，按"AI/前端/工具"分类。

把今天 Gmail 里收到的所有邮件分类：
紧急的 → 立刻 ping 我
工作类 → 总结要点
垃圾 → 直接归档
```

### 自动化执行
```
帮我打开携程，订一张本周五北京到上海的机票，
经济舱，10 点之后的，选最便宜的，付款用 Apple Pay。

每周五下午 4 点自动整理这周完成的工单，
按格式写成周报草稿发到我邮箱。
```

### 监控与告警
```
24 小时盯着我那 3 个 GitHub 仓库的 CI，失败立刻发钉钉。
顺便如果有新的 PR 提交，自动叫 Codex 帮我做第一轮 review。

Sentry 里出现等级 P0 的报错，5 秒内电话+短信+微信三路报警。
```

### 多人协作（团队 / 家庭）
```
家庭微信群里：
说"今晚买番茄"自动加进购物清单
说"老婆我下班了"自动同步到她的钉钉日历
每周日总结这周谁干了多少家务

公司飞书群里：
@机器人 问"上次那个客户的合同走到哪一步了"
机器人去 Notion 检索给出当前状态
```

### 也能写代码（联动 Codex / Claude Code）
```
在飞书里说一句：
"修一下 user-service 项目里那个登录超时的 bug，
跑通测试后提个 PR"

OpenClaw 会自动 clone 仓库、调用 Codex 或 Claude Code 去改、
跑测试、推分支、提 PR、在群里 @ 你贴上 PR 链接。
```

## 1.4 你能在哪里跟它对话

OpenClaw 的核心创意之一是：**不必学新软件**。它会出现在你已经在用的聊天工具里。

| 类别 | 支持的 channel |
|---|---|
| **国际主流** | WhatsApp · Telegram · Slack · Discord · iMessage · Signal · Microsoft Teams · Google Chat |
| **国内主流** | 飞书 · 钉钉 · 企业微信 · QQ · 微信 · WeChat (个人) |
| **开发者向** | IRC · Matrix · Mattermost · Synology Chat · Nostr · WebChat |
| **小语种 / 区域** | LINE（日韩台）· Zalo（越南）· Tlon · Twitch |

::: tip 跨 channel 协同
同一个 agent 可以同时在多个 channel 接收消息。比如你在家里用微信跟它说话，到公司在飞书继续，agent 的记忆是连贯的——这是 OpenClaw 的杀手特性之一。
:::

## 1.5 适用场景

**最适合用 OpenClaw 的人**
- 重度 IM 用户，每天泡在飞书/微信/Slack/Telegram 里
- 想做"长跑型"任务自动化（每天 / 每周自动执行某事）
- 对**数据隐私**敏感，不想把对话和工作内容交给云厂商
- 已有自己电脑或 NAS / 树莓派可以让它常驻
- 团队 / 家庭场景，需要"群里能用"的 AI

**不太适合的场景**
- 一次性问答（直接 ChatGPT 更简单）
- 重度纯编程工作流（Claude Code / Codex 更专业）
- 没有任何技术基础（OpenClaw 需要装 Node.js + 配 API Key，门槛比"打开网页"高）
- 想要"零配置开箱即用"的人（OpenClaw 是平台，需要你装配置）

::: warning 重要预期管理
OpenClaw **不是"装上就有 AI 助理"**。它是一个**框架**——你需要：
1. 给它一个 LLM API Key（决定它的"大脑"）
2. 配一个或多个 channel（决定怎么跟它对话）
3. 装/写 skill（决定它有哪些"手"和"工具"）

这三步装好后，它才真正能干活。本站接下来 6 章会一步步带你完成。
:::

## 1.6 安全模型

OpenClaw 默认运行在你自己的机器上，所有对话、文件、API Key 都在本地。但因为它能**执行系统命令、读写文件、控制浏览器**，权限非常大——这就像你把家钥匙交给了一个新雇的助理。

**官方建议**：
- 生产环境**不要直接跑在主力电脑**上，推荐隔离环境（VM、Docker、专用机器）
- 第三方 skill **当作不可信依赖对待**：用前必须看代码
- 每个 agent 只给**最小必要权限**
- 关键操作启用**二次确认**（如发钱、删数据）

这些会在 [15. 安全清单](/openclaw/ops/security-checklist) 一章详讲。
